# Database Configuration

## init.sql

Этот файл автоматически выполняется при первом запуске PostgreSQL контейнера (когда база данных пустая).

Он выполняет:
- Создание необходимых расширений (uuid-ossp, pg_trgm)
- Настройку привилегий для пользователя commit_user
- Оптимизацию настроек для разработки

**Важно**: Файл выполняется только один раз при создании базы данных. Если нужно пересоздать базу, удалите volume:
```bash
docker-compose down -v
docker-compose up
```

## postgresql.conf (опционально)

Файл с расширенными настройками PostgreSQL для production.

Чтобы использовать:
1. Раскомментируйте строки в docker-compose.yml для postgresql.conf
2. Перезапустите контейнер

## Структура

```
backend/database/
├── init.sql              # Автоматически выполняется при инициализации
├── postgresql.conf       # Опциональный конфиг для production
└── README.md            # Этот файл
```
